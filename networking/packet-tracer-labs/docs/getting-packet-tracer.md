# 🛠️ Getting Cisco Packet Tracer  

To run the labs in this repository, you’ll need **Cisco Packet Tracer** (recommended version: 8.x).  

---

## 📥 Download  
1. Go to the official **Cisco Networking Academy** site: [https://www.netacad.com/courses/packet-tracer](https://www.netacad.com/courses/packet-tracer)  
2. Create a free account (registration required).  
3. Download the latest Packet Tracer installer for your operating system (Windows, macOS, or Linux).  

---

## 💻 Installation  
- **Windows/macOS:** Run the installer and follow the setup wizard.  
- **Linux:** Install the `.deb` or `.rpm` package provided by Cisco.  
- Once installed, you can open `.pkt` files directly by double‑clicking them or using **File → Open** inside Packet Tracer.  

---

## 🚀 First Run  
- Launch Packet Tracer.  
- Open `pkt/topology.pkt` from this repo.  
- Explore the devices, configs, and verification steps in `docs/lab-instructions.md`.  

---

## 💡 Tips  
- Recommended version: **Packet Tracer 8.x** (older versions may not open `.pkt` files correctly).  
- Cisco provides short tutorials inside Networking Academy if you’re new to Packet Tracer.  
- If you run into issues, check the **screenshots/** and **results/** folders here for expected outputs.  

---

### ✅ You’re ready!  
With Packet Tracer installed, you can reproduce the labs, run verification commands, and experiment safely in your own environment.
